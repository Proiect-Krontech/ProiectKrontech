# Smart Pillow — Backend API

Backend REST API for the **Smart Pillow** project, an intelligent pillow that monitors the user’s posture in real time. Built with **Node.js + Express** and connected to **MongoDB** through Mongoose.

---

## Tech Stack

| Component | Technology |
|---|---|
| Runtime | Node.js |
| Framework | Express.js ^4.21.0 |
| Database | MongoDB (via Mongoose ^8.7.0) |
| Environment Variables | dotenv ^16.4.5 |
| CORS | cors ^2.8.5 |
| Dev Server | nodemon ^3.1.7 |

## Installation & Configuration

### 1. Clone the repository and install dependencies

```bash
cd backend
npm install
```

### 2. Configure environment variables

Copy the `.env.example` file and complete it:

```bash
cp .env.example .env
```

`.env` content:

```env
MONGODB_URI=mongodb://localhost:27017/smart-pillow
PORT=3000
```

### 3. Start the server

```bash
# Production
npm start

# Development (with auto-restart)
npm run dev
```

The server runs on `http://localhost:3000`.

## MongoDB Models

### `ActivationCode`

Each physical pillow has a unique activation code generated by the server.

| Field | Type | Details |
|---|---|---|
| `code` | String | Unique, indexed (e.g. `SP-AB3X7K`) |
| `pillowSize` | String (enum) | `S`, `M`, `L`, `XL` |
| `isActivated` | Boolean | Default `false` |
| `activatedAt` | Date | Set on first validation |
| `createdAt` | Date | Auto (timestamps) |
| `updatedAt` | Date | Auto (timestamps) |

The code is automatically generated in the format `SP-XXXXXX` using the characters `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (without ambiguous characters).

---

### `SensorData`

Stores each reading from the pillow’s pressure sensors.

| Field | Type | Details |
|---|---|---|
| `activationCode` | String | Reference to the pillow code, indexed |
| `sensors.fl` | Number | Front-left sensor |
| `sensors.fr` | Number | Front-right sensor |
| `sensors.bl` | Number | Back-left sensor |
| `sensors.br` | Number | Back-right sensor |
| `weight` | Number | Detected weight (kg) |
| `posture` | String (enum) | `correct`, `attention`, `wrong` |
| `score` | Number | Posture score 0–100 |
| `timestamp` | Date | Auto, indexed |

Compound index: `{ activationCode: 1, timestamp: -1 }` for fast per-device queries.

## API Endpoints

### Activation Codes — `/api/codes`

#### `POST /api/codes/generate`
Generates a new activation code for a pillow.

**Body:**
```json
{ "pillowSize": "M" }
```

**Response (201):**
```json
{
  "success": true,
  "code": "SP-AB3X7K",
  "message": "Activation code generated for pillow size M"
}
```

---

#### `POST /api/codes/validate`
Validates a code on first use and marks it as activated.

**Body:**
```json
{ "code": "SP-AB3X7K" }
```

**Response (200):**
```json
{
  "success": true,
  "code": "SP-AB3X7K",
  "pillowSize": "M",
  "message": "Code validated successfully. Welcome to your dashboard!"
}
```

---

### Arduino Communication — `/api/data`, `/api/status`, `/api/cmd`, `/api/command`

These endpoints handle real-time communication with the Arduino device integrated into the pillow.

#### `POST /api/data?code=SP-XXXXX`

The Arduino periodically sends sensor data. The server updates the in-memory state and stores it in MongoDB.

**Body (sent by Arduino):**
```json
{
  "posture": "good",
  "kg": 72.5,
  "F": 24.1,
  "S": 25.3,
  "L": 25.0,
  "R": 25.6,
  "buzzer": false,
  "calibrated": true,
  "weight_ref": 72.0,
  "state_duration": 120,
  "confidence": 0.95
}
```

Arduino posture values (`good` / `bad`) are mapped to the internal schema (`correct` / `wrong`).

---

#### `GET /api/status/:code`

Returns the current pillow status (online if it has sent data within the last 45 seconds).

**Response:**
```json
{
  "online": true,
  "posture": "correct",
  "kg": 72.5,
  "buzzer": false,
  "calibrated": true,
  "history": [ ... ]
}
```

---

#### `POST /api/cmd/:code`

Sends a command to the Arduino (retrieved on the next poll).

**Available commands:**

- `set_weight`
- `calibrate`
- `buzzer_on`
- `buzzer_off`
- `reset_calibration`

**Body:**
```json
{
  "cmd": "set_weight",
  "value": 75
}
```

---

#### `GET /api/command?code=SP-XXXXX`

The Arduino polls this endpoint to retrieve the pending command.

---

#### `GET /api/health`

Simple server health check.

**Response:**
```json
{
  "status": "ok",
  "timestamp": "2025-01-01T12:00:00.000Z"
}
```

## Posture Scoring Algorithm

The score is calculated based on the pressure distribution across the 4 sensors. The ideal distribution is equal (25% each).

```text
deviation = |fl - 25| + |fr - 25| + |bl - 25| + |br - 25|

score = max(
  0,
  round(100 - (deviation / 150) * 100)
)
```

| Score | Posture |
|---|---|
| ≥ 70 | `correct` |
| 40 – 69 | `attention` |
| < 40 | `wrong` |
